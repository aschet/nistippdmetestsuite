parserResErrors.txt

There is no parserResErrors.prg file.

This file contains what the NIST I++ response parser prints when the
parserResErrors.res file is run through it. The printed error messages are
native to the NIST response parser and are intended to describe the error
very specifically.

The parserResErrors.res file is designed to trigger all the source lines
of the form "errorCode = XXX;" in the order in which these lines appear in
the file parserRes.cc, so that is how they appear below. The second and
third digits of each tag below indicate the function that generated the
error. The third and fourth digits of each tag indicate the error message
within the function. For example, the tag 01308 means the following error
message is generated by the 13th function that generates error messages,
and it is the 8th error message generated by that function. Exceptions
to this tag numbering scheme are made made when the tag is required to be
E0000.

In addition to the numbering scheme, each time a new function is targeted,
a text string such as "start error messages from XYZfunction" has been
placed inside the response. In some cases this text string triggers the
error, but in most cases, the error is triggered (after the string has
been read) before the parser tries see if the string could be part of a
correct response.
